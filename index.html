懒是第一生产力 —— 使用GitHub Issue自动部署Github Pages

post @ 2023-04-12

目的

用最偷懒便捷的方式维护个人 blog，无需提交，编译，部署，维护。

使用效果

在自定仓库内提交 issue，标记 milestone 后，自动生成 Hexo 的新 Post，并自动编译部署到 Github Pages

仓库准备

  • 私有仓库yourname/blog：存放Hexo源文件，使用 issue 提交新 Post
  • 公有仓库yourname.github.io：存放Github Pages

源文件仓库构建

yourname/blog 按照存放Hexo文档构建并进行配置，**无需进行 depoly 相关配置**

添加 submodule

git submodule add git@github.com:yourname/yourname.github.io public

如果 public 目录非空，删除该目录再添加 submodule

配置 token

  • 在Github Token页面创建一个 token(classic)，给予 repo 相关权限，有效期选择永久，名字随便起一个，比如BLOG_TOKEN
  • yourname/blog`项目 Settings > Secrets and variables > Actions > New repository secrets > 填入刚创建的 token 和 name

配置 Hexo publish 工作流[1]

仓库 push 动作触发 Hexo 编译内容到 public 目录，并推送到 yourname.github.io 仓库进行 github pages 发布

yourname/blog项目 Actions > New workflow > set up a workflow yourself
随便给工作流起个名字，比如Publish

  • token: ${{ secrets.BLOG_TOKEN }} 中的 BLOG_TOKEN 替换为你自己的 token name
  • yourname 相关部分替换为你的用户名/仓库名

配置 Issue to Hexo 工作流[2]

提交 issue 并设置 milestone，触发gsongsong/issue-to-hexo,将本 issue 转换为 Hexo post 并提交

转换信息对应[3]：

GitHub Issue Hexo post Action
Title Title
Last updated date/time Date
Labels Tags
Body Content
Milestone Issue is converted only if milestone is set to publish

新建一个工作流，名为Issue_to_Hexo
